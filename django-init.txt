Вступ до Django

Django - Python фреймворк для веб-розробки.

Перед початком роботи пересвідчуємось у наявності python та його пакетного менеджера pip
PS C:...> pip --version
pip 25.2 from C:\Program Files\Python313\Lib\site-packages\pip (python 3.13)

Якщо програма не виконується, то
а) перевіряємо чи встановлений Python і чи є в нього pip. Якщо є, то потрібно або додати його шлях до системної змінної PATH, або зазначати повний шлях до pip при його запуску.
б) якщо Python немає, то встановлюємо його

Встановлюємо дозвіл на виконання скриптів (у PowerShell)
Set-ExecutionPolicy Unrestricted -Scope Process

Створюємо віртуальне оточення Python

Особливістю Python є те, що він встановлювався як єдина програма в операційній системі і тому всі модулі, що встановлюються додатково, суттєво впливають на роботу інших проєктів.
Для забезпечення можливості функціонування різних проєктів окремо, прийнято створювати віртуальне оточення.

Переходимо у директорію, вищу за корень проекту та вводимо команду

> python -m venv django

З'являється нова директорія django, яку відкриваємо як проект у редакторі (VSCode, PyCharm, ...)

---

---------------- При кожній новій роботі з проектом ----------------

Активуємо віртуальне оточення:

для PowerShell:

> ./Scripts/Activate.ps1

для Command Prompt

> ./Scripts/activate.bat

ознакою активації є зміна промпту із зазначенням назви оточення

(django) ...

Встановлюємо Django

python -m pip install Django
Після встановлення додається скрипт django-admin.exe через який буде здійснюватись подальше налаштування. Можна перевірити його роботу (успішність встановлення фреймворка)

(django)> django-admin --version

6.0.3
Структура Django складається з проектів та застосунків, для роботи необхідний принаймні один проект, у якому є принаймні один застосунок

(django)> django-admin startproject project_222

Додається директорія project_222 з головним скриптом управління проектом - manage.py, через який продовжується робота. Також створюється база даних (db.sqlite3)

Переходимо в директорію проекта та запускаємо сервер

(django)> cd project_222
(django)> python manage.py runserver

Зупиняємо сервер (Ctrl-C)
і створюємо застосунок

(django)> python manage.py startapp app222

Реалізуємо маршрутизацію

MVT - Model View Template

Model - постачальник даних

Template - шаблони відображення - заготовки веб-сторінок з підстановкою даних

View - функції, що відповідають за підготовку даних для шаблонів (Template)
|  Цей модуль аналогічний контролерам у MVC

Реалізуємо маршрутизацію.

app222/views.py - заходимо, редагуємо - створюємо функцію представлення

Створюємо файл внутрішньої маршрутизації: app222/views.py (див. файл)

Додаємо цей файл до проектної маршрутизації див. project_222/project_222/urls.py
(в ньому є інструкції - коментарі "з коробки", діємо за п. "Including another URLconf")

---------------- Template ----------------

Шаблони - HTML-сторінки з потенційною можливістю підстановки даних

створюємо директорію app222/templates
у ній створюємо файл home.html

у файлі app222/views створюємо функцію, що рендерить даний шаблон (див. файл)

Додаємо маршрут у файлі app222/urls.py

Реєструємо застосунок у налаштуваннях проекту project_222/settings.py :

INSTALLED_APPS = [
...
"app222"  --- додаємо цей рядок
]

Включаємо міграції

(django)> python manage.py migrate

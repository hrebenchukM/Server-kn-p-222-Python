## Розробка серверних додатків

### Інструкція з клонування проєктів Django
- Клонуємо проєкт або копіюємо код у директорію проєкту (django)
- Відкриваємо у терміналі директорію, вищу за проєкт Django (SERVER-...)
- Інсталюємо віртуальне оточення з назвою як у проєкту (django)
`python -m venv django`
Переконуємось, що у директорії проєкту (django/) виникли додаткові 
директорії, а поза нею інші директорії не створились
- Переходимо до директорії проєкту  `cd django`
- Активуємо віртуальне оточення, для PS команда `./Scripts/Activate.ps1`, переконуємось у зміні командного промпту (додається (django) на початку промпту)
- Встановлюємо модулі Django `pip install Django`, пересвідчуємось у тому, що модулі додались до директорії `Lib` даного проєкту. Якщо директорія лишається порожньою, то слід перевірити попередній пункт (чи активне віртуальне оточення)
- Переходимо до Django-проєкту `cd project_222`
- Переконуємось у правильності міграцій `python manage.py migrate`
- Запускаємо сервер проєкту `python manage.py runserver`, переходимо до браузеру

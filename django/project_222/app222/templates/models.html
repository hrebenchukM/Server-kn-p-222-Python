Моделі.

У Django, через відокремлення моделей форм та контексту, як моделі представлень, розуміння моделей зводиться до моделей даних (DTO, entities). Опис моделей передбачається у файлі models.py, який є частиною базового проєкту.

  • Оголошуємо клас-модель User у файлі models.py
  • Створюємо міграцію застосунку
    python manage.py makemigrations app222
    Migrations for 'app222':
    app222\migrations\0001_initial.py
    + Create model User
  • Фіксуємо міграцію
    python manage.py migrate
    Running migrations:
    Applying app222.0001_initial... OK

Для роботи з моделями існує потужний інструмент - адміністративна панель. Доступ до панелі потребує створення адміністратора (супер-юзера)
python manage.py createsuperuser

  • Запускаємо сервер (або переконуємось що він працює)
  • Переходимо за адресою /admin, використовуємо логін та пароль створеного супер-користувача
  • Реєструємо власні моделі для редагування в адмін.panelі
  • Відкриваємо розділ з моделлю User в адмін.panelі
  • Додаємо новий запис (користувача) через інтерфейс
  • Зберігаємо зміни та перевіряємо відображення даних
